# 1 Введение

<a name="appointment"/>

## 1.1 Назначение
В этом документе описаны функциональные и нефункциональные требования к веб-приложению **Cinema Management System** - системе управления кинотеатром для администраторов. Документ предназначен для команды разработки и тестирования приложения.

<a name="business_requirements"/>

## 1.2 Бизнес-требования

<a name="initial_data"/>

### 1.2.1 Исходные данные
Многие кинотеатры используют устаревшие системы управления или бумажные носители для планирования сеансов. Это приводит к ошибкам, накладкам и неэффективному использованию ресурсов.

<a name="business_opportunities"/>

### 1.2.2 Возможности бизнеса
Приложение позволит:
- Централизованно управлять всеми кинозалами и сеансами
- Минимизировать человеческие ошибки при планировании
- Оптимизировать загрузку залов
- Упростить процесс добавления и изменения расписания
- Вести историю всех изменений

<a name="project_boundary"/>

### 1.2.3 Границы проекта
**Cinema Management System** позволяет администраторам:
- Создавать, редактировать и удалять кинозалы
- Создавать, редактировать и удалять сеансы фильмов
- Просматривать полное расписание

Система не включает:
- Продажу билетов
- Управление пользователями (клиентами)
- Финансовую отчетность
- Онлайн-бронирование

<a name="analogues"/>

## 1.3 Аналоги
1. **Kinoteatr.ru** - коммерческая система управления кинотеатрами
2. **CinemaPlan** - планировщик для кинотеатров  
3. **Bitrix24** + модули для кинотеатров

**Преимущества Cinema Management System**:
- Простота и минималистичный интерфейс
- Быстрое развертывание
- Низкие требования к инфраструктуре
- Специализированность только на управлении залами и сеансами

<a name="user_requirements"/>

# 2 Требования пользователя

<a name="software_interfaces"/>

## 2.1 Программные интерфейсы
Приложение использует:
- Веб-браузеры (Chrome 90+, Firefox 88+, Edge 90+)
- База данных: PostgreSQL 14+
- Backend: Java Servlets, JSP
- Frontend: HTML5, CSS3, JavaScript (ES6+)
- Сервер приложений: Apache Tomcat 9+
- Сборка: Maven 3.6+

<a name="user_interface"/>

## 2.2 Интерфейс пользователя
Графический интерфейс включает:
1. **Главная панель** - обзор системы
2. **Раздел "Залы"**:
   - Таблица со списком всех залов
   - Кнопка "Добавить зал"
   - Кнопка удаления для каждого зала
3. **Раздел "Сеансы"**:
   - Таблица со списком всех сеансов
   - Форма добавления нового сеанса
   - Форма редактирования существующего сеанса
   - Кнопка удаления сеанса
4. **Навигационное меню** - переход между разделами

<a name="user_specifications"/>

## 2.3 Характеристики пользователей

<a name="application_audience"/>

### 2.3.1 Аудитория приложения

<a name="target_audience"/>

#### 2.3.1.1 Целевая аудитория
- Администраторы малых и средних кинотеатров
- Менеджеры киносетей
- Сотрудники, ответственные за составление расписания

<a name="collateral_audience"/>

#### 2.3.1.2 Побочная аудитория
- Технические специалисты кинотеатров
- Стажеры и обучающийся персонал

<a name="assumptions_and_dependencies"/>

## 2.4 Предположения и зависимости
1. Администратор имеет базовые навыки работы с веб-приложениями
2. Система работает в локальной сети кинотеатра или через VPN
3. Подключение к интернету требуется только для обновлений
4. Все пользователи системы проходят предварительное обучение

<a name="system_requirements"/>

# 3 Системные требования

<a name="functional_requirements"/>

## 3.1 Функциональные требования

<a name="main_functions"/>

### 3.1.1 Основные функции

<a name="hall_management"/>

#### 3.1.1.1 Управление кинозалами

| Функция | Требования |
|:---|:---|
| Просмотр списка залов | Отображение таблицы со всеми залами (ID, название, вместимость) |
| Добавление нового зала | Форма с полями: название (строка), вместимость (число). Валидация: название уникальное, вместимость > 0 |
| Удаление зала | Кнопка удаления с подтверждением. Если есть связанные сеансы - запрет удаления или каскадное удаление |
| Редактирование зала | Возможность изменить название и вместимость существующего зала |

<a name="session_management"/>

#### 3.1.1.2 Управление сеансами

| Функция | Требования |
|:---|:---|
| Просмотр списка сеансов | Таблица: фильм, зал, время начала, длительность, цена, действия |
| Добавление сеанса | Форма: выбор фильма (текст), выбор зала (выпадающий список), дата и время (datetime picker), длительность (минуты), цена (число). Валидация: нет пересечений по времени в одном зале |
| Редактирование сеанса | Изменение всех параметров существующего сеанса |
| Удаление сеанса | Удаление с подтверждением |
| Фильтрация сеансов | Фильтр по дате, залу |

<a name="view_schedule"/>

#### 3.1.1.3 Просмотр расписания

| Функция | Требования |
|:---|:---|
| Общее расписание | Календарный вид или таблица с группировкой по дням/залам |
| Расписание по залу | Просмотр всех сеансов конкретного зала на выбранный день |

<a name="restrictions_and_exclusions"/>

### 3.1.2 Ограничения и исключения
1. Нельзя удалить зал, если на него есть ссылки в сеансах (либо каскадное удаление)
2. Нельзя создать два сеанса в одном зале с пересекающимся временем
3. Время сеанса должно быть в будущем (нельзя создать сеанс в прошлом)
4. Максимальная длительность сеанса: 300 минут
5. Максимальная вместимость зала: 500 человек

<a name="non-functional_requirements"/>

## 3.2 Нефункциональные требования

- **Производительность**: время отклика ≤ 2 сек, поддержка 10+ пользователей
- **Надежность**: доступность ≥ 95%, ежедневные бэкапы
- **Безопасность**: защита от SQL-инъекций, валидация данных, логирование
- **Удобство**: интуитивный интерфейс, ≤3 кликов для основных операций
- **Поддерживаемость**: модульная архитектура, документация, расширяемость
